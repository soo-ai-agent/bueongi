# 스프린트 2 — 운영 안전 (Operational Safety)

> 목표: **키 남용·링크 유출·법규 리스크 차단.** 사고가 나면 되돌리기 어려운 항목을 선제 방어한다.
> 포함: P0-2, P2-11, P2-10

키를 저장소에 안 올린 것은 강점이지만, 운영 단계에는 별도의 리스크가 있다. 공개 키는 번들에서
추출 가능하고, 공유 URL은 로그인 없이 위치를 노출하며, 위치는 법적으로 민감정보다.

---

## P0-2. 클라이언트 키 남용·쿼터 거버넌스

- **배경/문제**: 카카오 JS 키·Tmap AppKey는 설계상 브라우저 번들에 인라인된다(공개 키라 정상,
  `clientEnv.ts` allowlist). 그러나 이는 **누구나 번들에서 키를 추출해 남의 쿼터를 소진**할 수
  있다는 뜻이다. Tmap은 호출당 과금 성격이라 비용 리스크가 직접적이다.
- **수정 내용**:
  - 카카오·Tmap 콘솔에 **도메인(referrer) 제한**을 등록한다(허용 오리진만 호출 가능).
  - 키별 **쿼터/요금 상한과 알림**을 설정하고, 초과 시 자동 폴백이 켜지도록 한다(→ P0-1 배지와 연동).
  - 키 **유출 시 회전(rotation) 절차**를 문서화한다.
  - **Tmap 경로 요청 캐싱**(동일 출발/도착 단기 재사용)을 확정한다
    (`mvp-orchestrator-design.md`의 "메모리/짧은 세션 캐시" 정책을 구현·검증).
- **완료 기준 (AC)**:
  - 운영 문서에 키별 도메인 제한·쿼터 상한·회전 절차가 명시된다.
  - 동일 출발/도착 재요청 시 캐시 히트로 Tmap 호출이 발생하지 않는다(테스트/로그 확인).
  - 쿼터 초과·키 오류 시 앱이 크래시 없이 폴백하고 P0-1 배지가 뜬다.
- **대상 영역**: `directRoute.ts`/`tmap.ts`(캐싱), 운영 문서, 외부 콘솔 설정(코드 외).
- **규모**: 소~중 · **의존성**: P0-1(폴백 표기)와 연동.

## P2-11. 공유 URL 유출 완화

- **배경/문제**: 공유 토큰만 알면 **로그인 없이** 위치를 조회한다(설계 원칙, `GuardianShare`).
  TTL 1~3h·귀가 완료 시 종료(`Navigation.tsx`, `ShareStatus.tsx`)로 완화돼 있으나, 링크가
  유출되면 만료 전까지 위치가 노출된다.
- **수정 내용**:
  - 공유 화면에 **수동 즉시 종료 / 재발급** 액션을 노출한다(현재는 귀가 완료 시에만 종료).
  - 공유 시점에 **"이 링크는 내 실시간 위치가 보입니다"** 경고 카피를 명시한다.
  - 종료 시 서버 토큰이 즉시 무효화되고 보호자 화면이 "공유 종료"로 전환됨을 보장한다
    (기존 `activeShare` / `ownerSecret` 흐름 활용).
- **완료 기준 (AC)**:
  - 공유 화면에서 즉시 종료·재발급이 동작하고, 종료 후 기존 링크는 만료 화면으로 귀결된다.
  - 공유 직전 위치 노출 경고가 노출된다.
  - 재발급 시 새 토큰이 발급되고 이전 토큰은 무효화된다.
- **대상 영역**: `ShareStatus.tsx`, `shareSession.ts`, 백엔드 `share/*`(무효화 경로 확인).
- **규모**: 소 · **의존성**: 백엔드 토큰 무효화 계약 확인.

## P2-10. 위치정보 법·규제 검토

- **배경/문제**: 위치는 민감 개인정보다. **위치정보법(위치정보의 보호 및 이용 등에 관한 법률)**
  상 신고·동의, 제3자(보호자) 제공 동의, 개인정보처리방침이 기획 산출물에 아직 없다. 시크릿
  위생과는 별개의 법무 트랙이다.
- **수정 내용**:
  - 개인위치정보 수집·이용·제3자(보호자) 제공에 대한 **동의 흐름 초안**을 설계한다
    (온보딩/최초 공유 시점 동의).
  - **개인정보처리방침 / 위치기반서비스 이용약관** 초안을 작성한다.
  - 위치정보사업/서비스 신고 필요 여부를 법무 검토 항목으로 등록한다.
- **완료 기준 (AC)**:
  - 동의 흐름·처리방침·약관 초안이 문서로 존재한다.
  - 법무 검토 대상 항목(신고 의무·보관 기간·파기)이 목록화된다.
  - 저장 정보는 좌표+만료 시각만(설계 원칙)이라는 점이 방침 문구와 일치한다.
- **대상 영역**: 기획/법무 문서, 온보딩·공유 동의 UI(후속 스프린트 연계).
- **규모**: 중 · **의존성**: P1-6(온보딩 동의)와 연계.
