# 스프린트 1 — 신뢰 방어선 (Trust Defense)

> 목표: **"속지 않는 안전 데이터"** 확보. 규모가 작고 임팩트가 큰 P0부터 처리한다.
> 포함: P0-1, P0-3, P0-4

안전앱의 신뢰는 "우리가 보여주는 안전 정보를 믿어도 되는가"에서 시작된다.
키가 env 주입 방식이라 **운영 중에도 폴백이 켜질 수 있고**(키 미설정 배포, 키 만료/회수,
쿼터 초과, 네트워크 장애), 이때 목업 안전점수가 무음으로 표시된다. 이 스프린트는 그 상태를
사용자가 반드시 인지하도록 만든다.

---

## P0-1. 라이브/폴백 출처 배지

- **배경/문제**: 외부 API 실패 시 CCTV·비상벨·파출소·안전점수까지 목업/로컬 카탈로그로
  **무음 폴백**한다(`requirements.md` AC3.2, `src/app/utils/placeSearch.ts` 등). 일반 UX엔
  좋지만, 사용자가 **샘플 안전점수를 실데이터로 착각**해 어두운 길을 선택할 위험이 있다.
- **수정 내용**:
  - 안전 데이터(안전점수, CCTV/비상벨/파출소 마커, 경로 옵션)에 데이터 출처 상태를
    `실데이터` / `예시(폴백)` 배지로 노출한다.
  - 폴백 판정은 각 데이터 소스 유틸(`routeSource.ts`, `policeSource.ts`, `cdnAssets.ts`,
    `placeSearch.ts`)의 폴백 분기에서 상태 플래그를 상단으로 전파한다.
  - "가짜 안내 금지" 원칙을 **데이터 출처 투명성**까지 확장한다.
- **완료 기준 (AC)**:
  - 키 미설정·쿼터 초과·네트워크 실패를 유발하면 해당 안전 데이터에 "예시 데이터" 배지가 뜬다.
  - 실데이터로 응답이 온 경우 배지가 "실데이터"이거나 배지가 없다(둘 중 택1, 일관 규칙).
  - 폴백 상태 전파에 대한 단위 테스트가 추가된다.
- **대상 영역**: `src/app/utils/*Source.ts`, `cdnAssets.ts`, `placeSearch.ts`,
  마커/경로 카드 컴포넌트(`RouteComparison.tsx`, `RouteDetail.tsx`, `map/RouteMap.tsx`).
- **규모**: 중 · **의존성**: 없음 (스프린트 최우선).

## P0-3. 데이터 신선도(갱신일) 노출

- **배경/문제**: CDN이 월1회/분기1회 갱신이다(`mvp-orchestrator-design.md` 캐시 정책).
  파출소 이전·CCTV 철거·비상벨 고장은 반영이 늦어 "이 근처 파출소/비상벨"을 100% 신뢰하면 위험.
- **수정 내용**:
  - `cdn/manifest.json`의 생성일·원천 정보를 앱이 읽어, 안전 데이터 상세에
    **"기준일 YYYY.MM + 출처"** 를 표기한다.
  - 자치구별 파일 선택(`cctv/{시군구}.json` 등) 시 해당 파일 기준일을 함께 노출한다.
- **완료 기준 (AC)**:
  - 마커 팝업/경로 상세/점수 상세에 기준일과 출처가 보인다.
  - manifest 로드 실패 시 "기준일 확인 불가"로 정직하게 표기한다(날조 금지).
- **대상 영역**: `cdnAssets.ts`(manifest 파싱), 마커 팝업/상세 컴포넌트.
- **규모**: 소 · **의존성**: P0-1과 표기 영역 공유(같이 처리 권장).

## P0-4. 안전점수 근거 설명

- **배경/문제**: 가중치(CCTV 0.35, 조명 0.25, 비상벨 0.20, 안심귀갓길 보너스 0.15 등,
  `safetyScore.ts` / 설계 의사코드)는 내부값이라 사용자는 "왜 이 경로가 안심인지" 모른다.
- **수정 내용**:
  - 점수 옆에 근거 요약을 노출한다. 예: "이 경로: CCTV 12 · 비상벨 3 · 안심귀갓길 60% 겹침".
  - **과신 방지 문구**를 함께 노출한다. 예: "실제 현장과 다를 수 있어요."
  - 근거 수치는 점수 계산에 쓰인 경로 버퍼(30m) 집계값을 그대로 재사용한다.
- **완료 기준 (AC)**:
  - 각 경로 옵션에 근거 항목·수치가 표시된다.
  - 과신 방지 문구가 안전점수 영역에 상시 노출된다.
  - 근거 수치가 점수 계산 입력과 일치함을 검증하는 테스트가 있다.
- **대상 영역**: `safetyScore.ts`(집계값 노출), `RouteComparison.tsx`, `RouteDetail.tsx`.
- **규모**: 소~중 · **의존성**: 없음.
