# 스프린트 3 — 여정 완성 (Journey Completion)

> 목표: **위급 시 실제로 작동하는 앱.** 핵심 사용자 여정의 공백을 메운다.
> 포함: P1-5, P1-6, P1-7, P1-8, P1-9
> 권장 순서: P1-6 → P1-7 → P1-8 → P1-5 → P1-9

현재 MVP는 사실상 "안전정보 지도 + 수동 위치 공유"에 가깝다. "안심귀가"의 정서적 핵심
(위급/미도착 대응)을 현재 스택으로 가능한 범위에서 채운다.

---

## P1-6. 온보딩 활성화 (권한 + 보호자 등록)

- **배경/문제**: `Onboarding`은 정적 안내뿐(AC1.1)이고, 정작 위급한 `Navigation`에서
  "보호자가 등록되어 있지 않아요" 분기를 만난다. **위급 순간에 등록을 요구하는 건 최악의 타이밍.**
- **수정 내용**:
  - 온보딩에 위치 권한 안내 + **보호자 최소 1명 등록** 단계를 추가한다.
  - P2-10의 위치정보 동의 흐름을 이 단계에 통합한다.
- **완료 기준 (AC)**:
  - 온보딩 완료 경로에 보호자 등록 진입점이 있고, 건너뛰기 시에도 이후 홈에서 재유도된다.
  - 보호자 등록 여부/시점이 계측된다(→ P2-14).
- **대상 영역**: `Onboarding.tsx`, `EmergencyContact.tsx`, `appStore`.
- **규모**: 중 · **의존성**: P2-10(동의).

---

## P1-7. 전역 SOS 접근성

- **배경/문제**: 112까지 경로가 `Navigation → 긴급 바텀시트 → 112`로 깊다. 홈/전역 상시 SOS가 없다.
- **수정 내용**:
  - 홈 및 주요 화면에 **상시 SOS**(112·보호자)를 노출한다.
  - 큰 탭타깃 + 오작동 방지(길게 누르기/확인 제스처)를 적용한다.
- **완료 기준 (AC)**:
  - 어느 주요 화면에서도 2탭 이내로 112에 도달한다.
  - 오작동 방지 제스처가 적용되고 접근성 라벨(aria-label)이 부여된다.
- **대상 영역**: `Layout.tsx`(전역 버튼), `Navigation.tsx`, 긴급 바텀시트.
- **규모**: 중 · **의존성**: 없음.

## P1-8. 112 위치 보조 제공

- **배경/문제**: `tel:112`는 통화만 연결한다. 신고자가 좌표를 말로 전해야 한다
  (`Navigation.tsx`의 긴급 플로우).
- **수정 내용**:
  - 긴급 시 현재 좌표를 **클립보드/문자로 보조 제공**한다.
  - "자동 전송"으로 날조하지 않고 "복사됨/문자 준비됨"으로 **정직하게** 표기한다(정직성 원칙).
- **완료 기준 (AC)**:
  - 긴급 바텀시트에서 현재 좌표 복사/문자 작성이 동작한다.
  - 카피가 실제 동작(복사/작성)과 일치하고 자동 전송을 암시하지 않는다.
- **대상 영역**: `Navigation.tsx` 긴급 바텀시트, `currentLocation.ts`.
- **규모**: 소 · **의존성**: P1-7(SOS 진입)와 같은 영역.

## P1-5. 미도착 안전망 (도착 예정시각 체크인)

- **배경/문제**: **경로 이탈 감지·미도착 알림·푸시**가 전부 v2 제외라, 미도착 상태를 사용자가
  놓칠 수 있다. 이는 "안심귀가"의 심장 기능이다.
- **수정 내용**:
  - 현재 스택으로 가능한 최소 안전망으로 **"도착 예정시각 체크인"** 을 도입한다.
  - 예정시각 경과 시 사용자에게 **보호자 공유(문자/카카오) 유도**를 띄운다(자동 푸시 없이, 정직 처리).
  - ETA는 기존 경로/`EtaBadge` 데이터를 재사용한다.
- **완료 기준 (AC)**:
  - 내비 시작 시 도착 예정시각이 설정되고, 경과 시 보호자 알림 유도 UI가 뜬다.
  - 자동 전송을 날조하지 않고 사용자 확인 후 공유로 이어진다.
- **대상 영역**: `Navigation.tsx`, `eta.ts`/`EtaBadge.tsx`, `share.ts`.
- **규모**: 중 · **의존성**: 보호자 등록(P1-6) 있으면 효과 상승.

## P1-9. 보호자 양방향 액션

- **배경/문제**: 사용자는 "보호자가 보고 있음"을 알 수 있으나(`Navigation.tsx`의 "보호자가
  함께 보고 있어요" — 좋은 기능), **보호자 → 사용자 방향 액션이 없다.** 보호자가 이상을 느껴도
  앱 안에서 할 수 있는 게 없다.
- **수정 내용**:
  - 보호자 공유 화면(`GuardianShare`)에 **"무사 확인 요청"** 등 최소 액션을 추가한다.
  - 요청이 발생하면 사용자 앱에 신호가 표시되도록 서버 계약을 확장한다(범위 최소화, v2 페어링과 구분).
- **완료 기준 (AC)**:
  - 보호자 화면에서 확인 요청 액션이 1개 이상 동작한다.
  - 사용자 앱이 요청을 인지할 수 있다(폴링/공유 서버 기존 채널 재사용).
- **대상 영역**: `GuardianShare.tsx`, 백엔드 `share/*`, `shareSession.ts`.
- **규모**: 중 · **의존성**: 백엔드 공유 서버 계약 확장.
